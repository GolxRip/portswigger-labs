**Status:** ✅ Completed (51/51)

## Overview

This learning path introduces SQL injection (SQLi), a critical web vulnerability. You learn how to detect and exploit SQLi to uncover hidden data and manipulate application behavior, as well as essential techniques to secure applications against SQLi attacks.

## Topics Covered

### 1. What is SQL Injection?
- Understanding SQL injection fundamentals
- Impact and severity

### 2. How to Detect SQL Injection Vulnerabilities
- Common injection points
- Detection techniques

### 3. Retrieving Hidden Data
- Using WHERE clauses to bypass logic
- Extracting unauthorized data

### 4. Subverting Application Logic
- Manipulating application behavior
- Authentication bypass techniques

### 5. SQL Injection UNION Attacks
- Determining number of columns
- Finding columns with useful data types
- Extracting interesting data
- Retrieving multiple values

### 6. Examining the Database
- Understanding database structure
- Information schema queries

### 7. Blind SQL Injection
- Triggering conditional responses
- Time-based exploitation
- Out-of-band (OAST) techniques

### 8. Error-based SQL Injection
- Leveraging database error messages
- Data extraction through errors

### 9. Second-order SQL Injection
- Stored injection attacks
- Deferred execution

### 10. Prevention Techniques
- Parameterized queries
- Input validation
- Prepared statements

## Key Takeaways

- SQL injection is one of the most critical web vulnerabilities
- Always use prepared statements and parameterized queries
- Implement strict input validation
- Monitor and log suspicious database queries
- Security testing should include SQLi checks
